什么是oracle

oracle数据库系统，是由美国甲骨文公司提供的以分布式数据库为核心的一组软件产品，oracle数据库也是一种关系数据库，此数据库体量较大，一般与jsp网站联合。oracle数据库弱化了库的概念，强化了用户的概念，即一个用户为一个库。

oracle系统表

oracle中内置了大量的系统表（有些类似于MySQL中的information_schema数据库），根据不同的权限可读取不同的系统表。可通过读取这些系统表获取我们想要的数据（用户信息、表、字段、数据库库信息等）。
DBA、ALL、USER、V_$、GV_$、SESSION、INDEX开头的大部分都是视图。
DBA_TABLES是DBA拥有的或可以访问的所有关系表。
ALL_TABLES是某一用户拥有的或可以访问的所有关系表。
USER_TABLES是某一用户所拥有的的所有关系表
要注意，在oracle数据库中对于数据来说，大小写是敏感的，所以在查询的时候，数据要区分大小写，代码可以不区分（表中数据默认大写）

oracle语法规则

在MySQL中可以直接使用select 1,2,3,后面可以不加from 表名，但是在oracle数据库中不行，它对于语法的完整性要求严格（有点像access数据库），当我们不知道表名的时候，可以使用dual代替，dual是oracle数据库的虚拟表，只是用来满足select的语法规则。

查询语句

查询所有的表select*from all_tables
查询当前用户的表select*from user_tables
查询所有的字段select*from all_columns
查询当前用户的字段select*from user_columns
查询版本select*from v$version

oracle中的rownum

oracle数据库中不支持MySQL中的limit功能，但是可以通过rownum来限制返回的结果集的行数（类似于MySQL中where id=1），rownum是由oracle数据库分配的，根据返回结果的数量来排序（不像MySQL中有固定的id值），默认从一开始排序。

当添加where rownum=1时，会返回一行数据。


当等于2时，页面报错了，在oracle数据库中不接受这样的语句，这时如果要显示两条数据，就要用小于号,以此类推

这时如果我们在查询字段的位置加上rownum，就会对其进行重新排序

oracle实现limit功能

1、利用不等于的方式获取
获取所有用户表select*from user_table

利用rownum让其只输出一条数据，现在输出的是NEWS表

在where条件后加and table_name<>’NEWS’，就可以获取第二条数据,意思是查询的时候，忽略这个已知表，从而达到获取其他表的目的，可以在后面加上多条and语句，来获取更多的表名

)
2、利用套嵌子查询的方式

select table_name from(select rownum a,table_name from user_tables) where a=1
其中select rownum a,table_name from user_tables是一个子查询
rownum a是给rownum起一个别名

那么我们将这条语句当做子查询嵌套进查询语句，并且加上where a=1，就可以得到第一条数据，where a=2，就可以得到第二条数据，依次类推
)

oracle显错注入

因为oracle的语法比较严格，所以需要用NULL,来填充字段

到了这一步，如果不知道字段的类型，很难进行下去。
在null的地方填充字符串类型，提示类型错误，所以，要么去一个个去试数据库的类型，要么就放弃显错注入

报错注入

报错注入，有的是利用更新失败使其报错（MySQL中的updatexml），
也有的是查询某些东西，没有查到使其报错。
CTXSYS.DRITHSX.SN这个函数咱也不知道是干啥的，反正只知道可以报错，以后知道再补上吧emmm!
语句：and 1=ctxsys.drithsx.sn(1,(select table_name from user_tables where rownum=1)),获得了第一个表名，剩下的表名，可以根据上面说的limit功能去获取
